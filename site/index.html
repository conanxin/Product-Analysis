    Source-first Workflow

    每篇 AI 辅助分析都经过以下工作流。核心原则：先 curl 实测 URL → 提取关键事实 → 显式标注 [事实] / [判断] / [推断]。

    1
    收集材料
    官方源 / 媒体 / Wikipedia / 投资人 portfolio
    2
    实链验证
    curl HTTP-200 测试，记录 verified / inaccessible
    3
    YAML metadata
    source_urls / review_status / source_url_verification_status
    4
    17 节完整
    包括 §17.1-§17.4 可信度分级与 Sources 表
    5
    人工复核
    draft → reviewed + partial / verified
    6
    索引同步
    article × index.yml × analyses/README.md × README.md 4 处一致
    7
    Validator
    scripts/verify_ai_analysis_index.py
    8
    GitHub Actions CI
    PR 必须 PASS validator，否则 CI 红灯

    Phase 1 Limits

    本仓库的诚实边界。

    • 所有文章仍是 partial，不是 verified；主体产品机制通常 verified，高风险事实 (融资 / 估值 / ARR / 用户数) 需 ≥ 2 个独立 verified 高质量来源才能 verified。
    • 私人公司高风险事实长期 partial：13 篇里 11 个是私人公司。
    • 不应把官方 blog / Wikipedia / Product Hunt / community posts 当作全部事实。
    • 主流媒体 paywall / 403 / 401 仍造成验证缺口：Reuters / Forbes / Bloomberg / CNBC / The Verge 等经常被防抓取或需订阅。
    • Cloudflare / WAF 拦截第三方产品页：Tola Capital / Lightspeed / Northzone / Product Hunt 等被锁。
    • 没有覆盖项目设计文档提到的"全部 14 维度"。
    • 本项目是研究档案，不是投资建议 / 采购建议 / 最终事实数据库。引用前请回原文核验 source_urls。
