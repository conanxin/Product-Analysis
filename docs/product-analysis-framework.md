# 产品分析框架

## 旧框架

早期人工分析使用的结构：

- **特点** — 产品最核心的特征
- **问题** — 观察到的问题或疑虑
- **设想** — 如果由我来做，会怎么改进

这个框架简洁高效，但维度有限，难以系统性输出。

---

## 新框架

从实践中提炼出 14 个分析维度：

| # | 维度 | 说明 |
|---|------|------|
| 1 | **一句话定位** | 用一句话说清楚这个产品是做什么的 |
| 2 | **目标用户** | 核心用户是谁？有什么特征？ |
| 3 | **使用场景** | 用户在什么时间、地点、状态下使用这个产品 |
| 4 | **核心用户路径** | 用户完成主要目标的步骤是什么？路径是否顺畅？ |
| 5 | **信息架构** | 内容/功能的组织方式？层级深浅？导航是否清晰？ |
| 6 | **交互设计** | 关键交互节点？有哪些值得学习的微交互？ |
| 7 | **内容机制** | 内容从哪里来？如何流转？谁来决定什么内容被看到？ |
| 8 | **社区机制** | 用户之间如何互动？有没有UGC？激励机制是什么？ |
| 9 | **增长机制** | 产品如何获取新用户？有没有病毒传播路径？ |
| 10 | **商业模式** | 如何赚钱？收入来源是什么？ |
| 11 | **竞品关系** | 直接竞品和替代方案有哪些？差异化在哪里？ |
| 12 | **产品问题** | 最大的体验问题或业务问题是什么？ |
| 13 | **如果我来重做** | 保留什么？改进什么？本土化机会？ |
| 14 | **对我自己项目的启发** | 这个产品对"如果我要做一个类似产品"有什么具体启发 |

## 分析维度表格

| 维度 | 要问的问题 | 输出形式 |
|------|-----------|----------|
| 一句话定位 | 这个产品解决什么人在什么场景下的什么问题？ | 1-2句话 |
| 目标用户 | 核心用户是谁？年龄/职业/使用动机？ | 2-4类用户画像 |
| 使用场景 | 什么时候用？在哪用？频率如何？ | 3-5个典型场景 |
| 核心用户路径 | 用户进来后做什么？关键步骤？ | 流程图/步骤列表 |
| 信息架构 | 内容如何组织？分类逻辑？导航结构？ | 站点地图/层级图 |
| 交互设计 | 关键交互？引导机制？容错设计？ | 3-5个交互细节分析 |
| 内容机制 | 内容来源？审核机制？推荐算法？ | 内容流转说明 |
| 社区机制 | 用户如何互动？奖励/惩罚机制？ | 社区机制图 |
| 增长机制 | 获客渠道？邀请机制？内容营销？ | 增长策略分析 |
| 商业模式 | 收入来源？定价策略？单位经济模型？ | 商业模式画布摘要 |
| 竞品关系 | 直接竞品？替代方案？差异化优势？ | 竞品对比表 |
| 产品问题 | 最大的3个问题是什么？严重程度？ | 问题列表+优先级 |
| 如果我来重做 | 保留/改进/新增什么？本土化机会？ | 具体改进建议 |
| 对自己的启发 | 3个可迁移的启发 | 具体启发点 |

## 为什么这个框架适合 AI 辅助分析

**AI 擅长的部分：**

- **资料整理** — 搜索产品背景、商业模式、竞品信息，整理成结构化文本
- **竞品归纳** — 快速梳理多个竞品的特点和差异，生成对比表格
- **结构化输出** — 按照指定格式输出分析内容，保持格式一致性
- **事实性描述** — 产品功能、界面、流程等客观描述
- **多维度覆盖** — 避免遗漏分析维度

**AI 不擅长的部分（需要人工复盘）：**

- **价值判断** — 某个设计决策到底是好还是不好，需要基于真实用户场景判断
- **优先级排序** — 产品问题那么多，哪个最值得投入资源解决？
- **迁移判断** — "对我有启发"这一点，必须结合自身项目实际情况
- **本土化分析** — 中国用户习惯、竞品格局、政策环境，AI 难以准确判断
- **隐藏动机** — 某些产品决策背后的商业逻辑，需要人工推测和验证

**推荐工作流：**

```
AI 输出初稿 → 人工快速复核 → 补充人工判断 → 定稿
     ↑                              ↓
  格式/事实检查              重点补充：问题判断+启发
```
