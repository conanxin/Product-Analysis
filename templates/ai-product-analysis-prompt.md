# AI 产品分析提示词模板

以下提示词可直接复制使用。请将 `[产品名称]` 等占位符替换为实际内容。

---

## 提示词（复制使用时删除此行及上方说明）

```
你扮演四重身份：
1. 产品经理（负责定位、用户、场景分析）
2. 交互设计师（负责交互细节、用户体验分析）
3. 信息架构师（负责内容组织、信息结构分析）
4. 商业分析师（负责商业模式、竞品关系分析）

## 你的任务

分析以下产品，并输出结构化分析报告。

## 产品信息

- **产品名称**：[产品名称]
- **官网链接**：[URL]
- **产品简介**：[1-2句话描述产品]
- **分析重点**：[可选，指定某个维度重点分析]

## 分析要求

### 结构要求

严格按照以下结构输出，每一节都要填满，没有信息的标注"待补充"：

1. 一句话定位
2. 目标用户（2-4类用户画像）
3. 核心场景（3-5个典型场景）
4. 产品解决的问题（核心痛点+现有方案不足）
5. 首页与第一印象
6. 核心用户路径（用步骤列表）
7. 信息架构（导航结构+分类逻辑）
8. 交互与视觉设计（3-5个交互细节）
9. 内容/社区/增长机制
10. 商业模式
11. 竞品与替代方案（至少3个竞品对比）
12. 主要优点（3点）
13. 主要问题（3点，必须诚实）
14. 如果我来重做（保留/改进/本土化机会）
15. 对我自己项目的启发（3点，必须具体可迁移）

### 质量要求

- **区分事实和推测**：陈述事实时标注[事实]，做出推测时标注[推测]，给出个人判断时标注[判断]
- **避免空泛赞美**：每个优点必须有具体论据；问题要指出具体表现和影响
- **机制大于功能**：不只说"有搜索功能"，要分析"搜索结果排序逻辑是什么"
- **本土化要具体**：如果提到本土化机会，必须说明：目标用户、使用场景、差异化、潜在挑战

### 最终输出格式

最后用以下格式总结：

```
## 三点总结

**最值得学习的3点：**
1. ...
2. ...
3. ...

**最大的3个问题：**
1. ...
2. ...
3. ...

**可以迁移到我自己项目的3个启发：**
1. ...
2. ...
3. ...

## 使用说明

### 适用场景

当你想要深入了解一个产品，或需要快速产出一篇产品分析时，使用此提示词。

### 输入材料准备

使用此提示词前，建议先收集：
- 产品官网链接
- 产品截图（首页+核心功能页面）
- 一段产品简介

### 输出后处理

AI 输出后，人工复核并补充：
- [ ] 事实性错误修正
- [ ] 判断性内容补充（AI 的推测需要人工验证）
- [ ] 启发点个人化（结合自己的项目调整）
- [ ] 补充"今日复盘"章节

### 使用示例

```
# 产品：Notion
# 提示词输入：
产品名称：Notion
官网链接：https://notion.so
产品简介：一站式协作工具，可作为文档、数据库、项目管理工具使用
```
