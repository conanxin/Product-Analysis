## URL 实链验证结果

### 验证范围

对 Perplexity 文章 Sources 区中 10 个主体质量 URL（官方+高质量媒体）使用 `web_fetch` 做 HTTP 实链验证。

### 结果

| URL | 类型 | HTTP 状态 | 验证结论 |
|-----|------|-----------|----------|
| https://www.perplexity.ai/ | primary-official | timeout | timeout-but-presumed（推定可达，CDN 保护） |
| https://www.perplexity.ai/enterprise | primary-official | timeout | timeout-but-presumed |
| https://aws.amazon.com/solutions/case-studies/perplexity-keynote-aws-reinvent-2023/ | primary-partner | **200** | **verified** ✅（标题匹配） |
| https://www.cnbc.com/2025/10/02/perplexity-comet-ai-browser-free-worldwide.html | high-quality-media (planned) | **404** | unverified (404) |
| https://www.cnbc.com/2025/09/03/paypal-venmo-perplexity-comet-ai-browser.html | high-quality-media (planned) | **404** | unverified (404) |
| https://www.cnbc.com/2026/01/30/perplexity-microsoft-azure-750-million-deal.html | high-quality-media (planned) | **404** | unverified (404) |
| https://www.macrumors.com/2026/05/08/perplexity-mac-personal-computer.html | high-quality-media (planned) | **522** | unverified (timeout) |
| https://www.theverge.com/2025/7/9/perplexity-comet-ai-browser.html | high-quality-media (planned) | **400** | unverified (invalid-path) |
| https://www.theinformation.com/articles/perplexity-200-billion-valuation | high-quality-media-paywalled | **403** | paywalled (Cloudflare) |
| https://www.reuters.com/technology/perplexity-new-york-times-lawsuit-2025-12-06/ | high-quality-media (planned) | **timeout** | unverified (timeout) |
| https://sourceforge.net/software/product/Perplexity-Pro/ | aggregator | **403** | blocked-by-Cloudflare |

### 验证汇总

- **verified (HTTP 200)**: 1 (AWS case study)
- **timeout-but-presumed**: 2 (perplexity.ai, /enterprise)
- **unverified (推定 URL)**: 6 (CNBC×3, Verge, MacRumors, Reuters)
- **paywalled (需订阅)**: 1 (The Information)
- **blocked (Cloudflare)**: 2 (SourceForge, The Information)

**核心结论**：P4 中"以高质量媒体为主来源"的声明是**高估**的。中文转载（new.qq.com / so.html5.qq.com）是**目前唯一被明确验证可访问的报道载体**。

## 新建 docs/source-quality-checklist.md

### 9 级来源分类

- primary-official
- primary-partner
- high-quality-media-verified
- high-quality-media-paywalled
- high-quality-media-inferred（P4.1 新增）
- secondary-syndication
- aggregator
- low-confidence
- unavailable

### 最低来源要求

- 至少 1 个 primary-official / primary-partner
- 至少 2 个 high-quality-media-verified
- 至少 1 个 secondary-syndication 补充
- 高风险事实双源验证

### draft → reviewed → verified 状态流

明确每一步的检查项与 YAML 字段。

### 常见错误案例

明确记录 P4 中出现的 4 个错误（推定 URL、未 HTTP 验证、声明与实际不符、可信度与实链脱节）作为复盘。
