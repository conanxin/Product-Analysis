## 6. 教训

P32 commit 6c210b5 的 commit message 显式声称"新增 ## 阶段性综合报告 入口"和"清理末尾行"，但实际上 diff 只显示 2 行 `[x] P31 / P32` 加入下一步计划列表 — README 主体结构未改。

**根因：** P32 编辑时我用 `edit` 工具的第一次合并和第二次 prepend 同时下达，但执行顺序混乱 — 先做的两条 `[x]` 行成功落地，后做的 `## 阶段性综合报告` 插入失败但未重试；末尾清理压根没发起。然后我立刻 `git commit -m` 描述中"诚实"写出意图，但实际未验证 diff。commit message 反映意图，不反映 diff，是 P32 的重大违规。

**应对（AGENTS.md Red Line #5 完成必须带证明）：**
- 下一次提交前必须 `git diff --cached --stat` 或 `git show <commit>` 与 commit message 对照
- 如 commit message 中声称"新增 X 入口"，diff 中必须能 grep 到 `^## X`
- 如 claim "清理"某行，diff 中必须能 grep 到 `-1, +1`

这条教训已记入 CHANGELOG.md 的 P32.1 章节"教训记录"小节。
