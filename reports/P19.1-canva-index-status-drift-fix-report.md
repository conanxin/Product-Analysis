## 7. 关键决策 (P19.1 Lessons)

### 7.1 决策模式

1. **P19.1 是 index-fix / docs-only 任务** — 不修改任何 AI 分析文章正文
2. **四处状态完全一致是核心目标** — Canva 文章 YAML / analyses/index.yml / README.md / analyses/README.md
3. **apply_patch 失败是漂移根因** — P19 升级时部分 edits 失败未察觉
4. **人工可读 vs 机器可读双索引** — 漂移发生在人工可读索引 (README.md)
5. **诚实评估优先** — 漂移发现后立即修复，不掩盖

### 7.2 P19.1 通用教训 (跨任务沉淀)

1. **apply_patch 失败需要 fallback** — P19.1 用 edit 单个替换作为 fallback
2. **四处索引必须同步** — Canva 文章 + 3 个索引文件
3. **漂移检测是 P 任务结束前的必查项** — 任何 review / index-sync 任务结束后应立即核对四处
4. **漂移修复不需要重新 P 报告** — 用 .1 子任务记录
5. **partial 状态跨文件必须一致** — 任何"reviewed"或"partial"标签漂移都应修复
6. **P19.1 子任务模式** — 当大任务出现次要漂移时，用 .1 子任务修复，不重做主任务

### 7.3 P19.1 与 P15-P19 经验集成

- P15 / P17 / P19 都涉及多文件索引同步
- 漂移是 P 任务的常见风险
- P19.1 子任务模式可以作为标准 P 任务结尾的"质量保证"步骤
- 后续 P 任务 (P20+) 应在 push 之前自动核对 4 文件状态一致性
