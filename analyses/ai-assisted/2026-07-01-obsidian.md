# Obsidian

## 1. 一句话定位

[事实] obsidian.md/ 官网首页标题为 "Sharpen your thinking"，副标题为 "The free and flexible app for your private thoughts"。首页顶部导航包括：Download / Pricing / Sync / Publish / Enterprise / Community / Account。官网多语言支持 12 种语言。

[事实] obsidian.md/about 是 JS-rendered，web_fetch 未能提取内容；obsidian.md/pricing 显示 Obsidian 采用 Catalyst → Standard → Commercial → Team → Enterprise 多层订阅模式。

[事实] obsidian.md/blog/future-of-plugins/（2026-05-12，由 kepano 发布）明确：Obsidian 由 Dynalist 团队开发（"people who created Dynalist"），核心定位是 "local-first" 和 "Markdown-based" 笔记与知识管理工具。

[判断] Obsidian 的核心产品定位不是"笔记软件"，而是"个人知识操作系统"（PKM — Personal Knowledge Management）。它的核心差异化在于：
1. **Local-first**：所有数据在本地 Markdown 文件，用户拥有数据
2. **Markdown-native**：纯文本格式，不依赖专有格式
3. **Graph-based knowledge base**：双向链接 + graph view 把笔记织成知识网络
4. **Highly extensible**：插件生态（4000+ 插件 / 120M downloads）和主题系统

Obsidian 不同于 Notion（云协作优先）和 Coda（doc-as-app 优先），它是 local-first + personal 优先，不主打团队协作而主打个人知识积累和深度思考。

[判断] Obsidian 不是 Notion 替代品，尽管两者在"笔记 + 双向链接"上有交集。Notion 是团队协作工具，Obsidian 是个人深度思考工具。

## 4. 产品解决的问题

[判断] Obsidian 解决的是 5 个相互关联的个人知识管理痛点：

1. **云笔记的隐私风险**：Notion / Evernote / 印象笔记等把数据存在云端，用户不拥有数据控制权；Obsidian 把数据存在本地，用户完全拥有。
2. **专有格式的锁定风险**：很多笔记软件用专有格式，数据迁移困难；Obsidian 用纯 Markdown，永恒可读。
3. **笔记之间的割裂**：传统文件夹式笔记只有树状结构，笔记之间没有横向联系；Obsidian 的双向链接让笔记形成网状。
4. **快速检索 vs 深度积累的张力**：传统笔记软件检索快但积累浅；Obsidian 的 graph view 和 backlinks 让积累越深越有价值。
5. **个人工具与团队工具的错配**：Notion / Coda 主打团队协作，但个人用户不需要协作成本；Obsidian 只服务个人，不强迫用户承担协作开销。

[事实] obsidian.md/blog/cure53-tob-sync-audits/（2026-05-13）提到 Obsidian Sync 于 2020 年上线，说明 Obsidian 的商业模式是"本地优先 + 可选云服务"的组合。

## 7. 信息架构

[判断] Obsidian 的核心信息架构：

| 对象 | 作用 | 层级 | 设计意义 |
|------|------|------|---------|
| **Vault** | 顶层容器，一个本地文件夹 | Vault | 用户完全拥有数据，迁移零成本 |
| **File（.md）** | 核心内容单元 | Vault > File | 纯 Markdown，不锁死格式 |
| **Internal Link [[]]** | 双向链接 | File > Link | 形成知识网络 |
| **Backlinks** | 反向链接面板 | File > Backlinks | 展示谁链接了我 |
| **Graph View** | 知识图谱可视化 | Vault > Graph | 全局看知识结构 |
| **Tag #** | 标签分类 | File > Tag | 横向组织 |
| **Plugin** | 功能扩展 | Vault > Plugin | 4000+ 可选扩展 |
| **Theme** | 视觉定制 | Vault > Theme | 1000+ 视觉主题 |
| **Sync** | 云同步服务 | 可选付费 | 跨设备同步 |
| **Publish** | 公开发布服务 | 可选付费 | 生成公开笔记站 |
| **Canvas** | 画布布局 | 可选功能 | 自由排版 |
| **Mobile** | 移动端 app | 可选 | iOS / Android |
| **Web Clipper** | 网页收藏 | 可选 | 剪藏网页内容 |

[判断] Obsidian 信息架构的核心特征是"Markdown 文件是一等公民"。不像 Notion 把 block/database 当作顶层，Obsidian 的顶层是文件，文件和文件之间通过链接形成网络。这种设计让 Obsidian 与文件系统天然融合，可以直接用任何 Markdown 工具编辑。

## 8. 交互与视觉设计

[事实] obsidian.md/ 展示的 Obsidian 视觉特征：
- 大标题 + 大留白
- 深色模式默认（obsidian 默认深色主题）
- 多语言支持（12 种语言）
- 社区 Discord / Twitter / YouTube 链接

[判断] Obsidian 交互设计的 7 个关键点：

1. **Local vault 是核心隐喻**：用户打开 vault 就是打开一个文件夹，文件管理器和笔记应用合二为一。
2. **双向链接 [[ ]] 是差异化核心**：输入 [[ 即可链接其他笔记，这种"写时链接"的体验是 Obsidian 区别于其他笔记软件的关键。
3. **Graph view 是知识发现的入口**：通过可视化知识网络，用户可以发现隐藏的关联。
4. **Backlinks 面板让引用透明**：每篇笔记可以看到谁引用了自己，形成知识上下文。
5. **社区插件生态是护城河**：4000+ 插件覆盖了从日程管理到数据可视化的各种需求，让 Obsidian 可以从"笔记工具"扩展成"个人 OS"。
6. **主题系统满足个性化**：1000+ 主题让 Obsidian 的视觉体验完全可定制。
7. **命令面板（Ctrl/Cmd + P）是效率入口**：快速访问所有命令、笔记、插件。

[判断] Obsidian 比 Notion 更"极客"的原因：Notion 面向协作设计，UX 偏向视觉化；Obsidian 面向个人深度思考，UX 偏向键盘操作和文本编辑。

1. **没有内置云同步（免费版）**：免费版没有云同步，多设备使用需要自己搭方案（iCloud / Dropbox / Git）
2. **Sync 服务需要单独订阅**：虽然买断便宜，但 Sync / Publish 是年订阅，长期成本累积
3. **插件质量参差不齐**：4000+ 插件中质量差异大，部分插件可能不再维护
4. **团队协作弱**：Obsidian 设计上不是团队工具，实时协作能力有限，与 Notion / Coda 比协作功能缺失
5. **学习曲线陡峭**：双向链接、graph view、插件系统对新手有门槛，"Zettelkasten 方法论"本身也需要学习
6. **安全性依赖本地设备**：所有笔记在本地，本地设备被破解则数据泄露；付费 Sync 服务有 Cure53 + Trail of Bits 审计但非银弹
