# Figma

> Figma 不只是"Sketch 的云端替代品",而是**以实时协作为核心、设计系统为骨干、Dev Mode 为桥梁、Config 2025 五大 AI 产品线为前沿**的公开公司级设计协作平台——把"画图"从个人工具重构为产品研发全流程的基础设施。

---

## 1. 一句话定位

Figma 是一款**云端实时协作的矢量图形编辑器与设计协作平台**,由 Figma, Inc. (NYSE: FIG) 开发。它不是简单的"在线版 Sketch",而是**围绕实时协作、设计系统、原型、Dev Mode 开发交付、Config 2025 新五大 AI 产品线(Sites/Make/Buzz/Draw/Weave)构建**的"以设计为入口的产品研发操作系统"。

[判断] Figma 的核心定位不是"更好用的画图工具",而是"**把设计工作流作为产品研发的中枢**"。这种定位决定了它的产品形态(从 canvas 到 AI 产品矩阵全部为协作 + 系统化服务),而不是传统图形软件。

## 2. 目标用户

[事实+判断] 根据 Figma 官网定位、Wikipedia 招股书数据、Config 2025 演讲和定价分层:

| 用户类型 | 典型特征 | 用 Figma 解决什么问题 |
|----------|----------|------------------------|
| **UI / UX 设计师** | 产品设计师、视觉设计师、UX researcher | 一站式 UI 设计、交互原型、设计系统维护 |
| **产品经理** | PM、Product Owner | 在 Figma 内直接评审设计、参与 review、导出 spec |
| **前端工程师** | Frontend Engineer | Dev Mode 导出 tokens / CSS / React / SwiftUI 代码 |
| **设计系统团队** | Design Ops、Design Systems lead | 维护 component library / variables / tokens |
| **市场 / 品牌团队** | Marketing、Brand Manager | Figma Buzz 品牌资产 / Slides 演示 / Make 营销页面 |
| **企业设计组织** | 1000+ 设计师 / 跨地域团队 | Enterprise 治理 (SSO/SCIM/audit log),统一品牌标准 |

[判断] Figma 的 PMF 核心是**前 4 类用户**,他们既是付费意愿最强的,也是与"协作"和"系统化"逻辑最匹配的;企业 / 品牌 / vibe design 是规模化收入来源。

## 4. 产品解决的问题

[事实+判断] Figma 解决的核心问题不是"画图软件不够好用",而是**"本地文件式设计流程与产品研发协作错位"**——传统设计工具是为"个人画图"设计的,而现代产品研发需要"团队协作 + 系统化 + 跨角色交付"。

**核心痛点:**

- **痛点 1:本地文件协作困难** — Sketch 文件(.sketch)需要版本管理、InVision + Zeplin + Dropbox 串联,流程碎片化
- **痛点 2:设计版本混乱** — 命名规则(主页_v2_final_真的final)无法解决多人协作
- **痛点 3:开发交付割裂** — 设计师用 Sketch/Figma,工程师用 Zeplin/Storybook,数据转换损耗大
- **痛点 4:设计系统难维护** — Component library 散落在多个文件,无法跨团队统一
- **痛点 5:设计师与工程师工具割裂** — 设计 review 与代码 review 分离,跨角色沟通成本高
- **痛点 6:远程团队难同步** — 异步反馈 + 多时区导致 review cycle 慢
- **痛点 7:AI 生成内容与设计工作流割裂** — ChatGPT / Midjourney 生成的内容需要手动导入,无法与设计系统联动

**Figma 的核心解法:**

- **云端实时协作** — 多人同编辑一个文件,multiplayer cursor + comment + @mention
- **设计系统作为一等公民** — Components / Variables / Tokens / Styles 系统化维护
- **Dev Mode 桥梁** — 设计师与工程师共享同一份 source of truth
- **Config 2025 五大 AI 产品线** — Make / Sites / Buzz / Draw / Weave 把 AI 嵌入设计工作流
- **Plugin 生态** — 1000+ plugins 扩展能力(Stark / Iconify / Unsplash / Chinese Fonts)

## 6. 核心用户路径

[判断] 用户从"首次访问"到"形成习惯"的典型路径:

```
注册账号 → 打开模板 (空白 / Community) → 创建 file → 邀请协作者 →
用 components 搭 UI → 评论 / 迭代 → Prototype 链接 →
Dev Mode 导出代码 → 继续迭代 → 升级 Pro / Organization
```

**企业用户路径:**
```
评估 → 申请 Enterprise 试用 → SSO 集成 → 创建 team / organization →
设计师建立 design system → 全员加入 → 跨部门 review →
admin dashboard 看使用情况 → 计费
```

**Config 2025 之后的新场景:**
```
设计师用 Make (prompt → UI) → 用 Sites (设计稿 → 网站) →
用 Buzz (prompt → 营销素材) → 用 Weave (image / video 编辑) →
design system 统一管理
```

**路径分析:**
- 步骤数:5 步可形成日活,8 步可形成付费
- 关键决策点:第一次邀请协作者 + multiplayer cursor 出现的瞬间
- 可能的流失点:网络延迟、企业 compliance、付费墙、复杂度

## 8. 交互与视觉设计

[判断] Figma 区别于"Sketch + Dropbox + Zeplin"的关键交互细节:

| 交互点 | 描述 | 评价 |
|--------|------|------|
| **云端实时协作** | multiplayer cursor + 实时同步,无版本冲突 | ✅ 区别于本地文件,无需版本管理 |
| **画布模型** | 无限画布 + Frame + Section,灵活布局 | ✅ 比 Sketch / XD 灵活 |
| **评论与 review** | 选中元素评论 + @mention + 状态(Open/Resolved) | ✅ 把 review 嵌入画布 |
| **Components / Variants** | 可复用 UI + 多种状态变体 | ✅ 设计系统基础 |
| **Variables (颜色/间距/Tokens)** | 全局变量 + Modes (Light/Dark) | ✅ 跨主题一键切换 |
| **Multiplayer cursor** | 多人光标 + 头像 + 当前操作 | ✅ 实时协作的"存在感" |
| **Prototyping flow** | Frame-to-frame 链接 + Smart Animate | ✅ 设计师直接做 prototype |
| **Dev handoff** | Inspect 面板 + CSS/iOS/Android 代码 | ✅ 设计师与工程师共享 source of truth |
| **Plugin workflow** | 1000+ 插件 + Figma Plugin API | ✅ 生态化扩展 |
| **AI 入口嵌入画布** | Figma Make (prompt → UI) 在画布直接运行 | ✅ "Vibe design" 入口 |
| **Auto Layout** | 容器自适应布局,响应式设计 | ✅ 区别于 Sketch,无需手动算位置 |
| **Design Tokens** | Variables → Code → 工程师用 | ✅ 设计师与工程师共享 design system |

[判断] 为什么 Figma 比"本地文件式设计工具"更像协作平台:
1. **云端文件 + 版本控制** — 不需要 Dropbox / Git LFS,所有人看到的是同一份
2. **Multiplayer cursor** — 实时"在同一个画布工作",建立团队存在感
3. **Comment + @mention** — 反馈嵌入画布,不用切到 Slack / 邮件
4. **Components / Variables** — 设计系统作为团队资产,而非个人文件
5. **Dev Mode** — 设计师与工程师共享 source of truth,减少切图流程
6. **Plugin 生态** — 1000+ 插件,生态化而非单点工具

**视觉风格:**
- 极简、白色背景为主
- 紫/蓝品牌色
- 大量使用"实时协作"动效
- Sans-serif 字体
- 大量 video demo 展示 AI / Config 2025 新功能

## 9. 内容 / 社区 / 增长机制

[事实+判断] 按 spec 需覆盖 7 个子项:

**1. Community 文件与模板**
- Figma Community (figma.com/community) 提供公开模板 / 组件 / 字体
- 用户可"remix"其他人的设计,降低学习成本
- 模板市场:UI kit / 移动端 / Web / 演示 / 数据可视化 / Icon 等

**2. Plugins / Widgets 生态**
- 1000+ plugins (Iconify / Stark / Unsplash / Chinese Fonts / Figma Ex)
- Plugin API 允许第三方扩展功能
- 是 Figma 平台化的核心

**3. 设计系统作为团队内传播机制**
- 设计师建立 design system → 全团队复用 → 品牌一致性
- 跨团队传播:"设计系统"从个人作品变成团队资产
- 是 Figma 在企业内部扩张的"病毒式"路径

**4. 教育 / 学生 / 设计师社区**
- Figma Education / Figma Students:免费提供给教育机构 / 学生
- Brown University 早期试点,Thiel Fellow 经历帮助早期扩张
- YouTube / Twitter / X 上大量设计师 demo 视频

**5. 组织内扩散:设计师 → PM → 工程师 → 企业管理**
